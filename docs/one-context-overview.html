Key Innovation

Feature Workspaces

A Feature is a named, persistent delivery workspace. It exists before a Jira Epic and creates one. Every role opens a Session — a fresh conversation that starts with a summary of everything discussed before.

🗂️

Persistent across roles

PO, Tech Lead, and Dev all work in the same Feature. No more "what did the PO decide?" — it's right there in the session history.

📌

Auto-linked artefacts

Every Jira story, Confluence page, and PR created inside a session is automatically linked to the Feature. OC-001 becomes the single source of truth.

⚡

Session briefs

Each new session starts with a generated brief of prior sessions — decisions made, artefacts created, open questions. No hunting through history.

Architecture

Four layers, one pipeline

Data flows from source systems through a hygiene pipeline into a vector knowledge store. The LangGraph agent retrieves from the store and writes back via confirmed actions.

Where we are on the roadmap

Five phases, each gated on the previous. No phase starts until the previous one proves value.

Phase 0 — Data Hygiene Pipeline

Clean, classify, and enrich before any ingestion

Complete
  • Jira + Confluence extractors (board-based)
  • LLM classifier + quality scorer (Claude Haiku)
  • Staleness detection + deduplication
  • Run against real data: 1,278 issues + 83 pages

Phase 1 — Grounded Assistant

Any team member asks questions grounded in real data

~85% Done
  • PostgreSQL + pgvector knowledge store
  • LangGraph agent (route → retrieve → reason → act → respond)
  • Live Jira + Confluence lookup tools
  • FastAPI + Celery + Docker Compose
  • Next.js Web UI (chat interface)
  • Validation gate: 10 questions vs ChatGPT

Phase 2 — Write + Remember

PO creates stories; team saves decisions; Feature workspaces

~90% Done
  • Story drafting flow with confirmation
  • create_jira_story, update_confluence_page
  • Team memory (remember tool + semantic search)
  • Feature workspaces — sessions, links, context injection
  • Validation gate: story creation time < 5 min

Phase 3 — Codebase Knowledge

The codebase is the bible — make it searchable for POs

~70% Done
  • Local git extractor (pathlib + gitpython, any host)
  • Code → business-language module summaries (Claude Haiku)
  • Git history as shipped_feature narrative
  • 14 repos ingested (ASX + Colleqt services)
  • Role-based retrieval weighting (PO vs Dev)
  • PR merge webhook → selective re-index
  • Validation gate: 10 PO questions from code only
